Introduction

Note : depending on the version you are using, this help file may contains differences between your version and the version it describes. The present help file is provided for YAPM 2.0.2.

Menu features

Here are the features available in the main menu (click on the Orb) and when you click on the small icons of the ribbon, near the Orb :

  • Change connection type (change the type of connection : local, remote via WMI or remote via server. See Remote management)
  • Emergency hotkeys (add custom hotkeys in case of emergency. For example, you can configure YAPM to kill the foreground application with Ctrl+Shift+Suppr shortcut. See Emergency hotkeys)
  • State Based Actions (add custom actions on processes. For example, you can configure YAPM to automatically reduce priority of a process if cpu usage is up to 50%. See State Based Actions)
  • Save report (save a report about the system)
  • Log file
  • System informations
  • Window list (displays the list of the windows opened by YAPM. You can show or close these windows)
  • Find Window' Process (See Find Window' Process)
  • Bug report & feedbacks (open a window with links to send feed back to me)
  • Hidden processes feature (show hidden processes, e.g. rootkits. See Show hidden processes)
  • Dependency viewer (show dependencies of a process/dll/... See Dependency Viewer)

Process features

The features available for process management are available in the 'Processes' tab of the main form. Use the icons on the ribbon, or right-click on the list. All actions are available for the different processes you select in the list.

  • Kill (it will terminate the process instantly)
  • Kill process tree (it will kill the selected process and all of its child processes)
  • Stop (pause the process, use "resume" to resume it)
  • Resume (resume the process after a "stop")
  • Change priority
  • Reduce working set size
  • Change affinity (select the processor(s) available for the process)
  • Create dump file (create a custom dump file of the process)
  • "Reanalize" (it will refresh fixed informations about a process, for example the path. It should be used when these informations are not available when the process has been added to the list. For example, if the path or the user name is not displayed, just reanalize the process to get the informations).
  • Show XXXX (it will show XXXX for the selected processes in the XXXX tab if the main form. For example, if you choose 'Show modules', all modules loaded by the selected processes will be displayed in the 'Modules' tab.)
  • File properties (open the Windows property dialog box about the executable)
  • Open directory (open the directory which contains the executable)
  • Show file details (show details of the executable in the 'File' tab)
  • Internet search (search for the executable name on the Internet)

Double-click on a process to open a detailed view of the process. See Detailed view for process.

Task features

The features available for task management are available in the 'Tasks' tab of the main form. Use the icons on the ribbon, or right-click on the list. All actions are available for the different tasks you select in the list.

  • Show window (show the main window of the task)
  • Maximize window (maximize the main window of the task)
  • Minimize window (minimize the main window of the task)
  • End task (end the task. It sends a 'close message' to the process, and tries to end it properly)
  • Select associated process (select the owner process in the 'Processes' tab)
  • Select window in window tab (select the owner window in the 'Windows' tab)

Double click on a task to select associated process.

Module features

The features available for module management are available in the 'Modules' tab of the main form. Use the icons on the ribbon, or right-click on the list. All actions are available for the different modules you select in the list. Note that this list is NOT automatically refreshed. To see the list of modules with auto refresh, you should open the detailed view of a process and select 'Modules' tab.

  • Select associated process (select the owner process in the 'Processes' tab)
  • Unload module (unload the selected modules from the memory of the process. You should NOT do it if you do not have a good reason !)
  • View dependency (show the dependencies of the module. See Dependency viewer)
  • File properties (open the Windows property dialog box about the executable)
  • Open directory (open the directory which contains the executable)
  • Show file details (show details of the executable in the 'File' tab)
  • Internet search (search for the executable name on the Internet)

Thread features

The features available for thread management are available in the 'Threads' tab of the main form. Use the icons on the ribbon, or right-click on the list. All actions are available for the different threads you select in the list. Note that this list is NOT automatically refreshed. To see the list of threads with auto refresh, you should open the detailed view of a process and select 'Threads' tab.

  • Select associated process (select the owner process in the 'Processes' tab)
  • Terminate (terminate instantly the thread)
  • Suspend (suspend the execution of a thread)
  • Resume (resume the thread after a "Suspend")
  • Change priority (change the priority of the thread)
  • Change affinity (change the affinity of the thread. Note that a thread can't use a processor if the owner process cannot use it)

Handle features

The features available for handle management are available in the 'Handles' tab of the main form. Use the icons on the ribbon, or right-click on the list. All actions are available for the different handles you select in the list. Note that this list is NOT automatically refreshed. To see the list of handles with auto refresh, you should open the detailed view of a process and select 'Handles' tab.

  • Select associated process (select the owner process in the 'Processes' tab)
  • Close handle (close the selected handles. You should NOT do it if you do not have a good reason !)
  • Display unnamed handles (display or not the handles which do not have name)

Window features

The features available for window management are available in the 'Windows' tab of the main form. Use the icons on the ribbon, or right-click on the list. All actions are available for the different windows you select in the list. Note that this list is NOT automatically refreshed. To see the list of windows with auto refresh, you should open the detailed view of a process and select 'Windows' tab.

  • Select associated process (select the owner process in the 'Processes' tab)
  • Find process (see Find process' feature)
  • Close (close the selected windows)
  • Position & size (change the position and/or the size of the selected windows)
  • Visibility -> Show (show selected windows)
  • Visibility -> Hide (hide selected windows)
  • Visibility -> Enable (enable selected windows)
  • Visibility -> Disable (disable selected windows)
  • Visibility -> Bring to front (bring the selected window to front)
  • Visibility -> Do not bring to front (do not bring the selected window to front)
  • Visibility -> Set as foreground window (set the selected window as foreground window)
  • Visibility -> Set as active window (set the selected window as active window)
  • Visibility -> Minimize (minimize selected windows)
  • Visibility -> Maximize (maximize selected windows)
  • Visibility -> Flash (flash selected windows : it will show an 'orange flash' in the Windows taskbar, if the window is present in the taskbar)
  • Visibility -> Stop flashing (stop the flashing of the selected windows)
  • Visibility -> Change caption (change the caption (text of the title bar) of the selected windows)
  • Visibility -> Change opacity (change the opacity. 0 is transparent, 255 is opaque).

Monitor feature

This feature allows to show the perfmon (Performance Counter) counters of Windows available in the system (local system if connection type is local, remote system if connection type is remote via WMI). Here are the actions available :

  • Add : add a counter. You will have to select a category, a counter and an instance (if required). You have to set the interval of refreshment
  • Remove selection : remove the counters selected in the tree
  • Start/Stop : start or stop the counters selected in the tree
  • Save report

To see a perfmon, select it in the tree. Then the associated graph will be displayed.

  • If you do not check 'Automatic' box, the left side of the graph will represent the date/time you select
  • If you check 'Now' box, the right side of the graph will represent the current date/time, else it will represent the date/time you select
  • If you check 'Automatic', or do not check 'Now', or check both, the number of values displayed will be determined by the number you specify (Max. values).

Service features

The features available for service management are available in the 'Services' tab of the main form. Use the icons on the ribbon, or right-click on the list. All actions are available for the different services you select in the list.

  • Stop (stop the selected services)
  • Start (start the selected services)
  • Pause (pause the selected services)
  • Resume (resume the selected services after a 'Pause')
  • Change start type (change the start type of a service. More types are available in detailed view)
  • Reanalyze (refresh some informations which are not automatically refreshed, for example the 'Start type')
  • File properties (open the Windows property dialog box about the executable)
  • Open directory (open the directory which contains the executable)
  • Show file details (show details of the executable in the 'File' tab)
  • Internet search (search for the executable name on the Internet)

File features

The features available for file management are available in the 'File' tab of the main form. Use the icons on the ribbon, or right-click on the list. The first thing to do is to open the file with 'Open file' icon. Then, some actions will be available :

  • Release (use it if your file appears to be 'locked', when you try to delete it for example. The processes which lock the file will be displayed)
  • Trash (delete the file, but keep it in the trash)
  • Delete (delete the file)
  • File properties (open the Windows property dialog box about the file)
  • Open directory (open the directory which contains the file)
  • Dir properties (open the Windows property dialog box about the directory which contains the file)
  • Internet search (search for the file name on the Internet)
  • Others -> Rename (rename the file)
  • Others -> Copy (copy to another directory)
  • Others -> Move (move to another directory)
  • Others -> Open (open with the associated application, or launch the executable)
  • Others -> Show file string (show the strings which can be read into the file. Note that for large file, it is not recommended to show file strings)
  • Others -> Encrypt (encrypt file using Windows encryption service)
  • Others -> Decrypt (decrypt file using Windows encryption service)

Double-click on a service to open a detailed view of the service. See Detailed view for service.

Search features

The search feature is available in the 'Search' tab of the main form.

You will have to specify which items you want to search (processes, modules, services, handles...). Then, enter the string you want to search, validate with 'Return' key and click on'Launch'.

You can close an item (i.e. terminate process for processes, unload module for modules, close handle for handles...) with 'Close item' menu.

Remote management

Change connection type

The first thing to do if you want to change the system you want to monitor is to change the connection type. There are three connection types :

  • Local : this is the default connection type. It is used to monitor the local system.
  • Remote via WMI : this is a connection type to monitor a remote system via WMI (Windows Management Instrumentation)
  • Remote via YAPM server : this is a connection type to monitor a remote system via a client-server architecture

To change the connection type, you have to open the 'Connection window' (Menu -> Change connection type). Then, disconnect from current machine, choose the new connection, configure it and click on 'Connect'.

Which connection should I choose ?

If you want to monitor the local machine, use 'local connection'.

 

If you want to monitor a remote machine, there are different points you should consider :

  • If you only want the list of processes/services with basic informations, Start/Stop features, you should use WMI
  • If you do not have an account on the remote machine, you have to use WMI
  • If you want to use Windows Performance Counters, you have to use WMI
  • If you want to monitor the remote system and manage handles, memory or use some other advanced features, you have to use the server
  • If you want to save report about the remote system, you have to use the server
  • If you want to search something with Search feature, you have to use the server
  • In other cases, keep in mind that :
  1. WMI might be fastest than the server
  2. The server should always work (even if WMI is not installed for example), this is not the case of WMI
  3. WMI might be easiest to use

WMI method

WMI method is the easiest way to monitor a remote system. You only need an account with a password on the remote machine. WMI must also be available in the remote machine (RPC server must be started). Here is the way you should follow to connect YYAPM to the remote machine :

  • start the remote machine and launch YAPM on the local machine
  • disconnect YAPM from the local machine
  • choose 'WMI connection'
  • enter the name of the remote machine
  • enter your username and your password (empty password are NOT allowed)
  • connect to remote machine ('Connect' button)

WMI is a nice way to monitor remote computers, but it is quite limited. Only some informations/features are available :

  • List of processes (but some informations are not visible)
  • List of services (but some informations are not visible)
  • List of modules (but some informations are not visible)
  • List of threads (but some informations are not visible)
  • New process / Kill process / Change process priority
  • All actions on services

Client-server method

Client-server method is more complex than WMI and it requires to run something (the server) on the remote machine. But it allows to monitor the processes/services/modules/... and all related objects just as if you were monitoring a local machine : all features/informations are available (except performance counters).

Here is the way you should follow to connect YAPM to the remote machine :

  • start the remote machine and launch YAPM on the local machine
  • start YAPM as a server on the remote machine (use launch server.bat, it launches YAPM with -server option)
  • disconnect YAPM from the local machine
  • choose 'Client-server connection'
  • enter the IP of the remote machine
  • connect to remote machine ('Connect' button)

Emergency hotkeys

This is a feature which allows to assign a shortcut to a specified action. Once you have open the 'Emergency hotkeys' window (Menu -> Emergency hotkeys), right-click on the listview and choose 'Add'. Then select the shortcut you want to use and the action you want to associate with. 'Remove' command delete the action definitevely (use Enable/Disable to temporarily remove a previous added shortcut). The shortcut are saved in 'hotkeys.xml' file in YAPM directory.

Note that the actions available depends from the version of YAPM you are using. For now, you can not add your own custom actions.

The shortcut will work only when YAPM is running.

Find Window' Process

This feature allows to find the process associated to a window displayed on the screen. Once you have open the 'Find Window' process' window (Menu -> "target" small icon), click on the form and drag your mouse over the screen. Select the desired window and release the left button of your mouse, it will select the process associated to the window.

Show hidden processes

This feature allows to view hidden processes, such as rootkits. As it uses user mode functions (no kernel functions), it shows only basic rootkits. There are two methods of detection available, you can change the method by clicking on the small "shield" icon on the bottom of the window.

Detailed view for process

When you double-click on a process (list of the processes on the main form), it opens a form which shows all available informations about the process. There are 16 tabs :

  • General : general informations about the process ('Get online infos' displays online informations about the process, depending on its name)
  • Statistics : all numerical informations available about the process
  • Performances : three performance graphs (Cpu usage with average cpu usage, physical memory usage, I/O delta)
  • Token : display privileges. Enable/Disable/Remove menus allows to enable/disable/remove the selected privileges
  • Memory : display all the memory regions of the process. 'View memory' menu opens the hex editor and shows the memory at the region' base address. 'Jump to peb' menu opens the hex editor and shows the memory at the PEB address.
  • Informations : display informations as text (RTF)
  • Services : display services of the process
  • Network : display TCP/UDP connections opened by the process
  • Strings : display string readable in the image (file on the disk) or in memory. It is possible to save the result and to search a sub string.
  • Environment : display all the environment variables of the process
  • Modules : display loaded modules. 'View memory' menu opens the hex editor and shows the memory at the module' base address.
  • Threads : display threads of the process
  • Windows : display windows of the process
  • Handles : display handles opened by the process
  • Log : powerful tool to monitor all changes made by/on a process. See Log feature.
  • History : display the history of one or more numerical counter(s). Select the desired counters on the left to display the associated graphs on the right. The history of all processes is collected in memory in real time and there is no need to open the detail view of the processes to collect informations, but the size of informations collected is limited. By default, only 100KB of history data are stored by process. You can configure it (or set it as unlimited) in the 'Preferences' window.

Detailed view for service

When you double-click on a service (list of the services on the main form), it opens a form which shows all available informations about the service. There are 4 tabs :

  • General - 1 : display general informations ('Get online infos' displays online informations about the service, depending on its name)
  • General - 2 : some other informations
  • Dependencies : display dependencies of the service
  • Informations : display informations as text (RTF)

Log feature

This feature allows to monitor all changes made by/on a process (Process detail view -> Log tab). Check 'Activate log' to enable the feature to collect the changes.

Click on 'Options' and choose the desired options :

  • Capture : the changes to monitor. For example, if you check 'Handles', it will monitor the handles opened/closed by the process. Check 'Created' and/or 'Deleted' to monitor new objects and/or deleted objects (in our example : opened handles and/or closes handles).
  • Show : the items to show on the list. If you uncheck all boxes, nothing will be displayed, and if you check all boxes, all collected changes will be displayed. This option should be used for two reasons : to see only specific changes (example : you monitor everything and just want to see new threads), and to economize CPU usage (when all options are activated, display all changes may require a lot of CPU time)
  • Interval : the interval between two 'snapshots of informations'. If you specify a small value, it may will require a lot of CPU time.
  • Auto scroll : automatically scroll (or not) the view (i.e. ensure that the last item of the listview is available).

Dependency viewer

This feature allows to view dependencies of an executable/*.dll/*.sys. If the Dependencies viewer is shown with Module->Show dependencies or Process->Show dependency, there is no need to select the file, otherwise you will have to use 'Open...' menu.

Once your file is opened, you will see the import/export tables and all informations available in a standard Dependency viewer.

Other features and tips

Here are some other features and tips you should know :

  • You should read the tooltips when you move your mouse over a control, there are sometimes some useful informations
  • 'Escape' key should close opened forms
  • Middle-mouse-click on a listview should copy the informations displayed for the selected items into the clipboard
  • Ctrl+Shift key on an item of a listview will display the pending tasks concerning the item.
