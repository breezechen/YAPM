Next release informations

Here are some informations about the next release.

The main changes of the next release will be :

  • Remote management via WMI : possibility to view processes/modules/threads/services of a remote machine (same workgroup) via WMI.
  • Remote management via the YAPM server : possibility to manage a remote machine (same workgroup) via a client-server architecture. It will be required to run the "server" executable on the remote machine.
  • Asynchronous operations : all actions (kill, enumerations, priority) will be asynchron. The goal is to never block YAPM, even if something is not responding (the remote machine for example).
  • Complete code refactoring for best performances (no more use of old API functions to enumerate processes & threads) and much less page faults
  • Completion of the State Based Action feature
  • New GUI with an "Orb"
  • New informations available for processes
  • Messages to warn user (dangerous actions and errors)
  • Bug fixes

Performance improvements : take ~10% less CPU usage than Process Explorer 11.30 (when both are minimized).

Progress

  • 2009-04-11 01:45 : After a hard work, WMI is now working (can get processes, services and modules, almost finished ~90%), and the client-server architecture WORKS TOO ! Only process enumerations is supported for now, but the new architecture with only asynchronous methods for remote management works :-)
